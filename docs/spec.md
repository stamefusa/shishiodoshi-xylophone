# 水駆動ししおどし楽器 システム仕様書

## 1. システム概要

MIDIキーボードの白鍵「ドレミファソラシド」を押すと、対応するウォーターポンプが一定時間作動し、各音に対応するししおどしへ水を送る。

ししおどしは水の重さで傾き、排水後に元の位置へ戻る。この動作によって、鉄琴または木琴の音板を叩いて発音する。

鍵盤入力から発音までの遅延は、この装置の演出として許容する。

## 2. 基本方針

システムは、次の役割分担とする。

### PC側

* MIDIキーボードからの入力を受信する
* MIDIノートとArduinoの出力ピンを対応付ける
* ポンプの作動時間を管理する
* 各音のクールダウン時間を管理する
* Arduinoへシリアルコマンドを送信する
* Arduinoからの応答を受信する
* 各音が入力可能かどうかを管理する

### Arduino側

* PCからシリアルコマンドを受信する
* 指定されたデジタル出力ピンを指定時間だけONにする
* 複数のポンプを同時に制御する
* 作動時間が経過したら、自動的に出力をOFFにする
* 不正なピン番号や時間を拒否する
* 緊急停止コマンドを受信したら、すべての出力をOFFにする

Arduino側では、MIDIノートとポンプの対応関係や、ポンプごとの作動時間を固定値として持たない。

これらの設定はPC側から毎回指定する。

## 3. システム構成

```text
MIDIキーボード
```

## 4. ハードウェア構成

### 4.1 使用機器

* USB MIDIキーボード
* PC
* Arduino Uno
* 8チャンネルMOSFETモジュール
* DC 3〜5Vウォーターポンプ × 8
* ポンプ用5V電源
* ししおどし機構 × 8
* 鉄琴または木琴の音板 × 8

### 4.2 ArduinoとMOSFETモジュールの接続

Arduinoのデジタル出力ピンからMOSFETモジュールの入力端子を直接制御する。

```text
Arduino D2 → MOSFET IN1
Arduino D3 → MOSFET IN2
Arduino D4 → MOSFET IN3
Arduino D5 → MOSFET IN4
Arduino D6 → MOSFET IN5
Arduino D7 → MOSFET IN6
Arduino D8 → MOSFET IN7
Arduino D9 → MOSFET IN8
```

ArduinoのD0およびD1は、USBシリアル通信で使用されるためポンプ制御には使用しない。

### 4.3 制御側電源

MOSFETモジュールの制御側端子は、次のように接続する。

```text
Arduino 5V  → MOSFETモジュール VCC
Arduino GND → MOSFETモジュール 制御側GND
```

### 4.4 ポンプ側電
```text
5Vポンプ電源 + → MOSFETモジュール VIN
5Vポンプ電源 - → MOSFETモジュール 負荷側GND
```

ポンプはMOSFETモジュールのOUT1〜OUT8へ接続する。

MOSFETモジュールが制御側と負荷側をオプトカプラで完全に絶縁している場合、Arduino側GNDとポンプ側GNDは接続しない。

実際の接続は、使用するMOSFETモジュールの端子表記と回路仕様を確認して決定する。

### 4.5 入力論理

MOSFETモジュールのH/Lジャンパーは、原則として次の設定を想定する。

```text
Arduino出力 HIGH → ポンプON
Arduino出力 LOW  → ポンプOFF
```

Arduinoコードでは、将来の反転に備えて以下の定数を用意する。

```cpp
constexpr uint8_t PUMP_ON_LEVEL = HIGH;
constexpr uint8_t PUMP_OFF_LEVEL = LOW;
```

実機確認で論理が逆だった場合は、この2つを入れ替える。

### 4.6 電気的な前提条件

Arduinoの出力ピンからMOSFETモジュールへ直接接続するため、MOSFETモジュールの各入力端子に必要な電流がArduinoの出力許容範囲内であることを確認する。

ポンプ側の「最大3A」という仕様は負荷側の出力電流であり、ArduinoからMOSFETモジュールへ流れる入力電流とは異なる。

最初は1チャンネルのみ接続し、正常にON/OFFできることと、ArduinoやMOSFETモジュールが異常発熱しないことを確認する。

## 5. 音階とピンの対応

PCクライアント側で、MIDIノート番号とArduinoピンを対応付ける。

初期設定は次の通りとする。

```text
MIDIノート60：ド       → Arduino D2
MIDIノート62：レ       → Arduino D3
MIDIノート64：ミ       → Arduino D4
MIDIノート65：ファ     → Arduino D5
MIDIノート67：ソ       → Arduino D6
MIDIノート69：ラ       → Arduino D7
MIDIノート71：シ       → Arduino D8
MIDIノート72：高いド   → Arduino D9
```

オクターブ名はソフトウェアによって表記が異なることがあるため、内部処理では音名ではなくMIDIノート番号を使用する。

黒鍵に対応するノート番号は無視する。

## 6. PC側設定

各音について、次の値をPC側で設定できるようにする。

* MIDIノート番号
* Arduinoの出力ピン番号
* ポンプ作動時間
* クールダウン時間
* 有効・無効

設定例：

```json
{
  "notes": [
    {
      "name": "C",
      "midiNote": 60,
      "arduinoPin": 2,
      "pumpDurationMs": 300,
      "cooldownMs": 5000,
      "enabled": true
    },
    {
      "name": "D",
      "midiNote": 62,
      "arduinoPin": 3,
      "pumpDurationMs": 320,
      "cooldownMs": 5000,
      "enabled": true
    },
    {
      "name": "E",
      "midiNote": 64,
      "arduinoPin": 4,
      "pumpDurationMs": 280,
      "cooldownMs": 4800,
      "enabled": true
    }
  ]
}
```

ポンプごとの流量差やししおどしの個体差は、`pumpDurationMs`を変更して調整する。

設定変更だけではArduinoの再ビルドや再書き込みを必要としない。

## 7. PCクライアントの状態管理

各音は、PC側で次の状態を持つ。

```text
READY
ACTIVE
COOLDOWN
ERROR
```

### 7.1 READY

鍵盤入力を受け付けられる状態。

対応する鍵盤が押されたら、Arduinoへポンプ作動コマンドを送信する。

### 7.2 ACTIVE

Arduinoがポンプを作動させている状態。

同じ音の再入力は受け付けない。

別の音の入力は受け付ける。

### 7.3 COOLDOWN

ポンプ停止後、ししおどしが排水・復帰するまで待機している状態。

同じ音の再入力は受け付けない。

設定されたクールダウン時間が経過したらREADYへ戻る。

### 7.4 ERROR

Arduinoとの通信失敗や不正な応答が発生した状態。

ユーザーによるリセット操作、再接続、または状態確認によって復旧する。

## 8. MIDI入力仕様

PCクライアントはMIDIのNote Onイベントを受信する。

次の条件を満たした場合に、ポンプ作動処理を開始する。

* 対象ノートが設定に登録されている
* 対象ノートが有効である
* 対象ノートの状態がREADYである
* Note Onのベロシティが1以上である

Note Offイベントは使用しない。

ベロシティ0のNote OnはNote Offとして扱い、無視する。

初期実装ではベロシティによるポンプ作動時間の変更は行わない。

## 9. PCとArduinoの通信仕様

### 9.1 通信方式

* USBシリアル通信
* 通信速度：115200bps
* ASCIIテキスト
* 1コマンド1行
* 行末：LF（`\n`）
* CRLF（`\r\n`）も受信可能とする

### 9.2 ポンプ作動コマンド

```text
PUMP <pin> <durationMs>
```

例：

```text
PUMP 2 300
```

意味：

```text
Arduinoのデジタルピン2を300ミリ秒間ONにする
```

### 9.3 個別停止コマンド

```text
STOP <pin>
```

例：

```text
STOP 2
```

指定したピンを即座にOFFにする。

### 9.4 全停止コマンド

```text
STOP_ALL
```

すべてのポンプ出力を即座にOFFにする。

緊急停止、PCアプリ終了、シリアル切断前などに使用する。

### 9.5 状態確認コマンド

```text
STATUS
```

Arduinoは許可された全ピンの現在状態を返す。

## 10. ArduinoからPCへの応答

### 10.1 起動完了

```text
READY
```

Arduinoの初期化と全出力OFFが完了した後に送信する。

### 10.2 ポンプ作動受付

```text
ACK PUMP <pin> <durationMs>
```

例：

```text
ACK PUMP 2 300
```

PCはこの応答を受信したら、対象音をACTIVE状態にする。

### 10.3 ポンプ停止完了

```text
DONE <pin>
```

例：

```text
DONE 2
```

PCはこの応答を受信したら、対象音をCOOLDOWN状態にする。

### 10.4 使用中

```text
BUSY <pin>
```

指定ピンがすでにONの場合に返す。

PCは同じポンプを重複作動させない。

### 10.5 停止受付

```text
ACK STOP <pin>
```

または、

```text
ACK STOP_ALL
```

### 10.6 不正なピン番号

```text
ERROR INVALID_PIN <pin>
```

### 10.7 不正な作動時間

```text
ERROR INVALID_DURATION <durationMs>
```

### 10.8 不正なコマンド

```text
ERROR INVALID_COMMAND
```

### 10.9 入力形式エラー

```text
ERROR INVALID_FORMAT
```

### 10.10 状態応答

`STATUS`に対して、次の形式で各ピンの状態を返す。

```text
STATUS 2 OFF
STATUS 3 ON
STATUS 4 OFF
STATUS 5 OFF
STATUS 6 OFF
STATUS 7 OFF
STATUS 8 OFF
STATUS 9 OFF
STATUS_END
```

## 11. Arduino側の制御仕様

### 11.1 使用可能ピン

PCからピン番号を指定できるが、Arduinoはすべてのピンを無条件には受け付けない。

許可するピンは次の8本に限定する。

```text
D2
D3
D4
D5
D6
D7
D8
D9
```

これにより、シリアル通信用のD0、D1や、意図しないピンの誤操作を防止する。

### 11.2 作動時間

PCからミリ秒単位で指定する。

許可範囲は次の通りとする。

```text
最小：10ms
最大：5000ms
```

5000msを超える指定は拒否する。

上限はポンプの連続作動や水漏れを防ぐための安全制限であり、PC側から変更できない固定値とする。

### 11.3 複数ポンプの同時作動

異なるピンであれば、複数のポンプを同時にONにできる。

例：

```text
PUMP 2 300
PUMP 4 500
PUMP 7 250
```

上記3つは、それぞれ独立した終了時刻を持つ。

### 11.4 同一ピンの重複作動

すでにONになっているピンへ再度`PUMP`コマンドが来た場合は、現在の終了時刻を変更せず、次を返す。

```text
BUSY <pin>
```

作動時間の延長や再スタートは行わない。

### 11.5 ノンブロッキング制御

Arduinoコードでは`delay()`を使用しない。

各ピンについて、次の情報を保持する。

* 現在ONかOFFか
* 作動開始時刻
* 作動時間

`millis()`を使用して、各ポンプの停止時刻を監視する。

これにより、ポンプ作動中でもシリアルコマンドの受信と、ほかのポンプの制御を継続する。

### 11.6 `millis()`のオーバーフロー

時間判定は、次のような差分比較を使用する。

```cpp
if (static_cast<uint32_t>(millis() - startedAt) >= durationMs) {
    // ポンプ停止
}
```

絶対時刻同士の単純比較は使用しない。

## 12. Arduino起動時の安全動作

Arduinoの起動時またはリセット時は、次の順序で初期化する。

1. 許可ピンD2〜D9をOUTPUTに設定する
2. すべての許可ピンをポンプOFFレベルに設定する
3. 各ピンの内部状態をOFFに初期化する
4. シリアル通信を開始する
5. 初期化完了後に`READY`を送信する

Arduinoが起動しただけでポンプが動かないことを最優先とする。

## 13. シリアル受信処理

固定長の文字配列を使って1行ずつ受信する。

Arduino Unoのメモリを考慮し、`String`クラスは使用しないか、少なくとも多用しない。

要件は次の通り。

* LFでコマンド確定
* CRは無視する
* 受信中もポンプ停止時刻の監視を継続する
* バッファ長を超えた行は破棄する
* 不完全なコマンドで出力を変更しない
* 数値変換に失敗した場合はエラーを返す
* 負数や極端に大きい数値を受け付けない
* 未定義コマンドでピン状態を変更しない

## 14. PCクライアントの画面要件

最低限、次の情報を表示する。

* 接続中のMIDI入力デバイス
* 接続中のArduinoシリアルポート
* Arduino接続状態
* 各音のMIDIノート番号
* 各音のArduinoピン番号
* 各音のポンプ作動時間
* 各音のクールダウン時間
* 各音の現在状態
* 通信ログ
* 緊急停止ボタン

各音の設定は、PCクライアントの画面または設定ファイルから変更できるようにする。

## 15. PCクライアントの動作フロー

```text
MIDI鍵盤が押される
        ↓
対応する設定を検索
        ↓
対象音がREADYか確認
        ↓
Arduinoへ PUMP pin duration を送信
        ↓
ACK受信
        ↓
対象音をACTIVEへ変更
        ↓
Arduinoがポンプを停止
        ↓
DONE受信
        ↓
対象音をCOOLDOWNへ変更
        ↓
設定されたクールダウン時間を待つ
        ↓
READYへ戻す
```

Arduinoから`BUSY`または`ERROR`が返された場合は、対象音をACTIVEにしない。

## 16. 通信異常時の処理

### 16.1 ArduinoからACKが返らない場合

一定時間以内にACKが返らなければ、対象音をERROR状態にする。

初期値として、ACK待機時間は1000msとする。

### 16.2 シリアル切断時

PCクライアントは、切断前に可能であれば`STOP_ALL`を送信する。

ただし、突然のUSB切断でも各ポンプはArduino側のタイマーによって必ず自動停止する。

### 16.3 PCクライアント終了時

終了処理で`STOP_ALL`を送信してからシリアルポートを閉じる。

### 16.4 Arduino再起動時

Arduinoから`READY`を受信したら、PC側の全音状態をREADYへ初期化する。

必要に応じて`STATUS`を送信し、実際の出力状態を確認する。

## 17. センサーについて

初期版では、ししおどしの転倒や復帰を検知するセンサーは使用しない。

クールダウン時間はPC側で固定時間として管理する。

## 18. 初期設定値

初期試作では次の値から開始する。

```text
ポンプ作動時間：300ms
クールダウン時間：5000ms
Arduino通信速度：115200bps
ACK待機時間：1000ms
Arduino側最大作動時間：5000ms
```

ポンプ作動時間とクールダウン時間は、各ししおどしの動作結果を見ながらPC側で個別に調整する。

## 19. 受け入れ条件

### Arduino単体

* 起動時に全ポンプがOFFである
* `PUMP 2 300`でD2が約300msだけONになる
* 作動終了後に`DONE 2`を返す
* 異なるピンを同時に作動できる
* 同じピンの重複作動を拒否できる
* D2〜D9以外を拒否できる
* 5000msを超える作動時間を拒否できる
* `STOP 2`でD2を停止できる
* `STOP_ALL`ですべて停止できる
* 長時間動作しても`millis()`を使った制御が停止しない
* 不正な文字列を受信してもポンプが誤作動しない

### システム全体

* MIDIノート60を押すとD2のポンプが作動する
* 黒鍵を押してもポンプが作動しない
* MIDIノートごとに異なる作動時間を指定できる
* 複数の白鍵を押した場合、対応するポンプが同時に作動する
* COOLDOWN中の音は再作動しない
* 別の音はCOOLDOWNに関係なく作動できる
* PC側設定変更だけでピン番号と作動時間を変更できる
* 緊急停止ですべてのポンプを停止できる
